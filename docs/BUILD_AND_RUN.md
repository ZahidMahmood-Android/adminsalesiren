# Flutter Admin Panel - Build & Run Configuration

## Quick Start

### Prerequisites
- Flutter SDK installed (https://flutter.dev/docs/get-started/install)
- Chrome browser (for web development)
- Dart Code extension installed in VS Code

### Running the Project

**Option 1: VS Code Task (Recommended)**
1. Press `Cmd + Shift + B` (or `Ctrl + Shift + B` on Linux)
2. Select **Flutter: Run Web Dev Server** or **Flutter: Run Web Chrome**
3. The app will open automatically in your browser

**Option 2: Terminal**
```bash
# Development server (useful for development)
flutter run -d web-server

# Chrome browser (interactive debugging)
flutter run -d chrome
```

**Option 3: Debugging in VS Code**
1. Press `F5` or go to Run > Start Debugging
2. Select the desired launch configuration:
   - **Flutter Web (Debug)** - Standard debug build
   - **Flutter Web (Chrome - Verbose)** - Debug with verbose logs
   - **Flutter Web (Release)** - Optimized release build

## Available VS Code Tasks

| Task | Purpose |
|------|---------|
| Flutter: Run Web Dev Server | Hot reload development server |
| Flutter: Run Web Chrome | Launch in Chrome browser |
| Flutter: Build Web Release | Create optimized production build |
| Flutter: Build Web Debug | Create debug build |
| Flutter: Clean | Clean build artifacts |
| Flutter: Pub Get | Fetch dependencies |
| Flutter: Analyze | Run code analysis |
| Dart: Format Code | Format code according to Dart conventions |

Access these via **Terminal > Run Task...** or `Cmd + Shift + P` → "Run Task"

## Development Workflow

1. **First Time Setup:**
   ```bash
   flutter pub get
   ```

2. **Run Development Server:**
   - Use `Cmd + Shift + B` to start dev server
   - Code changes auto-reload in browser

3. **Code Quality:**
   - Format code: Run **Dart: Format Code** task
   - Analyze: Run **Flutter: Analyze** task before committing

4. **Building for Production:**
   - Add a `## YYYY-MM-DD` entry to [`docs/release-management.md`](./release-management.md) (version, target URL, summary).
   - Run **Flutter: Build Web Release** task or:
     ```bash
     chmod +x tool/build_web_release.sh
     ./tool/build_web_release.sh
     ```
   - Output: `build/web/` directory (gitignored — deploy only, never commit).

5. **Before pushing to Git:**
   - Run **Flutter: Analyze** — fix warnings.
   - Do not commit `build/`, `docs/keys-and-secrets.md`, `.env`, `functions/node_modules/`, or `.firebase/`.

## Troubleshooting

**Chrome not found**
```bash
flutter config --enable-web
flutter devices  # Check available devices
```

**Port already in use**
```bash
# Kill process on port 8080
lsof -ti:8080 | xargs kill -9
```

**Clean rebuild**
```bash
flutter clean
flutter pub get
flutter run -d chrome
```

## Firebase Configuration

The project uses Firebase. Ensure:
- `lib/firebase_options.dart` exists (generated by FlutterFire CLI)
- Firebase credentials are set up in your Firebase project
- Web API keys are configured in Firebase console
